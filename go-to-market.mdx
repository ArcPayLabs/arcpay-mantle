ArcPay's first market is Mantle builders who need agent payments, policy controls, privacy-intent payments, and proof-ready demos.

## Target Users

- AI-agent developers building paid services
- early Mantle teams that need agent commerce primitives
- operators managing contributor or agent budgets
- developer relations teams looking for compelling Mantle demos
- early teams that need invoices, spend controls, and audit evidence

## Wedge

Start with developer adoption:

1. Publish the x402 starter kit.
2. Promote the HTTP developer tools and MCP server.
3. Show a 3-minute demo of a paid agent endpoint unlocking after Mantle escrow.
4. Offer privacy intents and USDY cards as reusable components.
5. Convert builders into workspace users when they need policy and audit controls.

## Acquisition Channels

- Mantle Discord developer channels
- Mantle builder and agent developer groups
- X demo clips showing x402, privacy intents, and USDY cards
- GitHub starter-kit distribution
- Mintlify docs and OpenAPI reference
- direct outreach to teams building agents, games, DeFi bots, or paid data endpoints on Mantle

## 30-Day Plan

- Week 1: publish docs, demo video, starter kit, and developer API examples.
- Week 2: onboard 5 Mantle builders into the x402 starter kit and collect feedback.
- Week 3: ship `AgentReputationBook` and reputation UI for agent discovery.
- Week 4: publish a builder case study and apply for Mantle ecosystem support.

## 90-Day Plan

- 25 developer signups through docs, CLI, or HTTP tools.
- 10 live agent endpoints registered or tested.
- 3 teams using x402 or privacy-intent examples.
- 1 public integration partner or grant conversation.
- Production hardening for hosted developer tools and workspace auth.

## 12-Month Direction

ArcPay becomes the financial operations layer for agent businesses on Mantle:

- agent service marketplace
- on-chain reputation and settlement proofs
- policy automation based on agent performance
- privacy-intent SDK
- hosted x402 gateway
- team workspaces with audit exports and billing controls
