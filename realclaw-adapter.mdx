ArcPay Mantle includes a RealClaw handoff layer for operators who want a Telegram-controlled agent to coordinate approved Mantle treasury work.

RealClaw's Mantle campaign surface uses Mantle Skills and venue activity across Fluxion, Merchant Moe, and Agni Finance. ArcPay treats RealClaw as an agent execution/control surface and keeps treasury policy, x402 payments, escrow, privacy intents, invoices, reputation, and audit records inside ArcPay.

We also reviewed `byreal-git/byreal-sdk`. That package is a Solana DEX SDK: it imports `@solana/web3.js`, exposes Solana CLMM pool/position/swap/copy-farming helpers, and uses Solana mint/pool addresses. It is useful for understanding Byreal's DEX product surface, but it is not a Mantle execution SDK. ArcPay Mantle therefore does not pretend to execute Mantle actions through that SDK.

The boundary is intentional:

- RealClaw owns the Telegram bot token, Telegram agent setup, Mantle Skills workflow, and operator chat surface.
- ArcPay owns treasury policy, x402 payment requirements, order escrow, privacy intents, USDY cards, invoices, reputation, and audit records.
- The agent receives a structured handoff payload instead of unrestricted wallet authority.

## Operator Flow

1. Create and configure the Telegram agent inside RealClaw with BotFather.
2. Connect the wallet RealClaw registers as your Mantle agent address.
3. Keep the bot token inside RealClaw. Do not paste it into ArcPay.
4. Open ArcPay `/realclaw`.
5. Set the strategy name, RealClaw agent address, budget, allowed assets, venue targets, risk limits, x402 resource, and executor slug.
6. Copy the handoff payload into the RealClaw Telegram agent instructions/config.
7. Route planning through ArcPay policy and x402, execute via RealClaw-supported Mantle venues, then attach tx/volume/ROI evidence back into ArcPay.

## Handoff Payload

ArcPay generates an agent-readable payload containing:

- Mantle Testnet network metadata.
- RealClaw agent address placeholder.
- Fluxion, Merchant Moe, and Agni venue allowlist.
- ArcPay x402 protected resource URL.
- AgentRegistry, AgentOrderBook, TreasuryPolicy, PrivacyVault, and ReputationBook addresses.
- Budget and asset constraints.
- Operator approval and leverage restrictions.
- Evidence requirement before any fill, yield move, or paid work is marked complete.

## What Requires RealClaw Setup

RealClaw execution requires the operator to complete RealClaw's Telegram onboarding and connect the agent wallet RealClaw registers for Mantle activity.

ArcPay does not store the Telegram token. After RealClaw is configured, the operator can paste the ArcPay payload into the Telegram agent and use ArcPay as the Mantle policy/payment/evidence layer.

Direct venue execution must come from RealClaw/Mantle Skills, Mantle venue contracts, or an operator wallet flow. The Solana Byreal SDK should not be used as a Mantle executor.

## Safety Model

ArcPay should not grant unrestricted spend authority to any external agent. Use:

- `TreasuryPolicy` for budgets and policy.
- `AgentOrderBook` for escrowed work.
- x402 for paid HTTP resources.
- `MantlePrivacyVault` for commitment-based payment intents.
- `AgentReputationBook` for service outcome history.
- `/audit` and `/status` for evidence.
