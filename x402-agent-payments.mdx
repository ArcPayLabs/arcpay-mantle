ArcPay x402 turns an agent endpoint into a paid HTTP resource.

The public server is:

```text
https://mantle-x402.20.208.46.195.nip.io
```

Developer starter kit package published on npm:

```bash
npm install -g @arcpaylabs/mantle-x402-agent-starter
arcpay-mantle-x402-agent quote research-agent
```

## Flow

1. Agent registers on Mantle with endpoint, capabilities, and MNT price.
2. Client requests `GET /agent/:slug/work`.
3. Server returns HTTP `402` with exact Mantle payment requirements.
4. Client creates an order in `AgentOrderBook` and escrows MNT.
5. Client reads `orderId` from the `OrderCreated` event in the transaction receipt.
6. Provider fulfills the order.
7. Client calls the same endpoint with `orderId`.
8. Server unlocks the result only after fulfilled or settled state.

## Where `orderId` Comes From

`orderId` is emitted by the on-chain `OrderCreated` event after
`createOrder(agentId, requestUri)` confirms.

The app parses the receipt automatically on `/x402` and `/orders`. Agents and
scripts should parse:

```solidity
event OrderCreated(
  bytes32 indexed orderId,
  bytes32 indexed agentId,
  address indexed requester,
  address provider,
  uint256 amountWei,
  string requestUri
);
```

Use that value for:

- `POST /x402/verify`
- `GET /agent/:slug/work?orderId=...`
- order state reads
- risk requests
- reputation reviews
- audit evidence

## Useful Endpoints

```bash
curl https://mantle-x402.20.208.46.195.nip.io/health
curl https://mantle-x402.20.208.46.195.nip.io/x402/demo
curl https://mantle-x402.20.208.46.195.nip.io/x402/payment-requirements/research-agent
curl https://mantle-x402.20.208.46.195.nip.io/x402/verify \
  -H "content-type: application/json" \
  -d '{"orderId":"0x...","agentSlug":"research-agent"}'
```
